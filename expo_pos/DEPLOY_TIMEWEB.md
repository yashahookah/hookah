# Expo POS → Timeweb: пошаговый деплой

Используем репозиторий **yashahookah/hookah** (expo_pos в подпапке). В корне репо есть `Dockerfile` для сборки.

---

## Шаг 1. Зайти в Timeweb Cloud

1. Откройте [timeweb.cloud](https://timeweb.cloud)
2. Войдите в аккаунт (или зарегистрируйтесь)
3. Пополните баланс при необходимости (деплой платный)

---

## Шаг 2. Создать приложение в App Platform

1. В меню слева: **Сервисы** → **App Platform**
2. Нажмите **«Создать приложение»**
3. Или сразу: [timeweb.cloud/my/apps/create](https://timeweb.cloud/my/apps/create)

---

## Шаг 3. Выбрать тип приложения

1. Выберите **Backend**
2. Выберите **Dockerfile** (не FastAPI — у нас приложение в подпапке)
3. Нажмите **Далее**

---

## Шаг 4. Подключить GitHub

1. В блоке **«Репозиторий»** нажмите **«Добавить аккаунт»** (если GitHub ещё не подключён)
2. Авторизуйте Timeweb в GitHub
3. Выберите репозиторий: **yashahookah/hookah**
4. Ветка: **main**
5. Нажмите **Далее**

---

## Шаг 5. Настроить сборку (если есть поля)

- **Путь к Dockerfile:** `Dockerfile` (в корне репо)
- **Контекст сборки:** корень репозитория (по умолчанию)

Если интерфейс упрощённый — оставьте значения по умолчанию.

---

## Шаг 6. Запустить деплой

1. Нажмите **«Запустить деплой»** / **«Создать»**
2. Дождитесь сборки (5–10 минут)
3. В логах не должно быть ошибок

---

## Шаг 7. Получить ссылку

После успешного деплоя появится публичный URL, например:

```
https://ваш-проект-xxxxx.timeweb.cloud
```

---

## Шаг 8. Проверить работу

Откройте в браузере:

| Страница   | URL                                      |
|-----------|-------------------------------------------|
| Киоск     | `https://ваш-проект-xxxxx.timeweb.cloud/kiosk`   |
| Продавец  | `https://ваш-проект-xxxxx.timeweb.cloud/seller`  |
| Сборка    | `https://ваш-проект-xxxxx.timeweb.cloud/picking`  |

Главная `/` перенаправляет на `/kiosk`.

---

## Если в интерфейсе нет «Dockerfile»

Тогда используйте **FastAPI**, но нужен отдельный репозиторий с `expo_pos` в корне:

1. На GitHub: **New repository** → имя `expo-pos`
2. На компьютере:
   ```bash
   cd /Users/bws/expo_pos
   git init
   git add .
   git commit -m "Expo POS"
   git remote add origin https://github.com/yashahookah/expo-pos.git
   git push -u origin main
   ```
3. В Timeweb подключаете репозиторий **yashahookah/expo-pos**
4. Тип: **FastAPI**, файл — `main.py`

---

## Важно про SQLite

На App Platform файловая система может быть временной. Данные SQLite могут пропадать при перезапуске. Для выставки на 1–2 дня этого обычно достаточно. Для постоянной работы лучше использовать VPS (см. полную инструкцию в этом файле выше).
